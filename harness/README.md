# Investment Idea Harness

金融投資アイデアを **検証可能な単位** に分解し、
「着想 → 仮説 → データ検証 → バックテスト → リスク評価 → 実行 → 振り返り」
を反復するためのハーネス構造です。

## フォルダ構造

- `01_idea_backlog/` : 生アイデア、テーマ、観察メモ
- `02_hypothesis/` : 検証可能な仮説（条件・KPI・棄却条件）
- `03_dataset/` : 取得データ、加工済みデータ、データ辞書
- `04_signal_research/` : シグナル定義、特徴量、探索ノート
- `05_backtest/` : 検証コード、期間別結果、感度分析
- `06_risk/` : VaR、ドローダウン、ポジション設計、制限値
- `07_execution/` : 執行ルール、発注手順、チェックリスト
- `08_journal/` : 日次トレードログ、例外記録
- `09_review/` : 週次/月次レビュー、改善アクション
- `templates/` : 仮説・検証・レビューのテンプレート

## 運用ルール（最小）

1. アイデアは `01_idea_backlog` に必ず記録してID採番（例: `IDEA-2025-001`）
2. 実行前に `02_hypothesis` で仮説を明文化
3. 仮説1件につき最低1回 `05_backtest` を実施
4. `06_risk` の制約を満たさない案は実行しない
5. 実行後は `08_journal` → `09_review` で振り返る
